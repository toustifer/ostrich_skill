# Diary Writer

三层日记体系：**日 → 周 → 月**，全部存放在 `D:\obi\collected_program\日记\YYYY\MM\` 下。

## 三层结构

| 层级 | 文件名 | 触发词 |
|------|--------|--------|
| 日 | `YYYY-MM-DD.md` | 日记、写日记、补日记、今天 |
| 周 | `WXX-周结.md` | 周结、本周、这周 |
| 月 | `月结.md` | 月结、本月、这个月 |

## 日日记

### Workflow

1. 确定目标日期。有指定日期用指定的，否则用今天。
2. 检查文件是否存在。不存在则按模板创建。
3. 追加内容到对应栏目，保留已有内容。

### 默认模板

```
# YYYY-MM-DD 日记
- 日期：YYYY-MM-DD
- 星期：周X
- 天气：
- 心情：
## 今日记录
## 想法与感受
## 感谢
## 明日待办
```

## 周结

### 触发

用户说”周结””写周结””本周总结”等。

### 文件位置

`日记/YYYY/MM/WXX-周结.md`，XX 为 ISO 周号。覆盖日期范围写在标题中，如 `W22 周结 — 2026-05-25 ~ 2026-05-31`。

### 模板

```
# WXX 周结 — YYYY-MM-DD ~ YYYY-MM-DD
## 本周目标
## 完成事项
## 未完成 / 遗留下来
## 核心收获
## 数据（可选）
## 下周重点
```

### 生成规则

- 自动从本周的日日记中汇总”完成”和”未完成”的待办
- 核心收获从日日记的”想法与感受”中提炼
- 如果本周日日记数据不足，提醒用户口述补充

## 月结

### 触发

用户说”月结””写月结””本月总结”等。

### 文件位置

`日记/YYYY/MM/月结.md`。

### 模板

```
# YYYY 年 M 月 月结
## 本月关键事件
## 项目进度总览
## 数据复盘
## 下月重点
```

### 生成规则

- 项目进度从所有子项目 README 和周结中汇总
- 数据复盘从日/周数据中聚合
- 如果跨多个项目，提醒用户确认哪些项目纳入月结

## Writing rules

- 口语化的、自然的日记散文风格，除非用户要求原始笔记
- 用户多次追加内容时，继续追加同一文件，不要新建
- 补日记用过去日期
- 只说”写日记”则先创建今天文件，再填充
- 每次写完日记后，提醒用户是否需要同步更新周结或月结
