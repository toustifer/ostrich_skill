# Learning Vault Spec

本文件是 learning-project-summary 技能的 source of truth。所有学习项目卡片均按此规范创建和更新。

## 根目录规则

- 所有项目都放在 `学习/` 下
- 一个项目一个文件夹
- 文件夹名直接使用项目名
- 项目根目录必须有一个与文件夹同名的索引文件

```text
学习/
  项目名/
    项目名.md
```

## 标准目录结构

每个项目默认生成以下结构：

```text
项目名/
  项目名.md
  学习目标/
    学习目标.md
    目标A/
      目标A.md
    目标B/
      目标B.md
```

- `项目名.md`：项目总览页
- `学习目标/学习目标.md`：目标索引页
- `学习目标/目标名/目标名.md`：单个目标详情页

## 命名规则

- 目录名 = 页面主标题 = 同名 md 文件名
- 不使用英文 slug，不额外附加编号
- 默认直接使用中文业务名或能力名
- 不创建 README、说明书、安装指南等额外文件

## 项目总览页模板

文件路径：`项目名/项目名.md`

```md
# {项目名}

## 项目描述
{一句到两句，说明这是个什么项目、代码在哪里、你打算通过它学什么}

## 创建日期
{YYYY-MM-DD}

## 项目状态
进行中

## 个人动机
{为什么要做这个项目}

## 能力预期
{做完后希望获得什么能力}

## 薄弱环节
{当前最明显的短板，1-3 条即可}
```

- 总览页只写高层信息
- 不写太长的技术细节
- 不展开成详细教程

## 学习目标索引页模板

文件路径：`项目名/学习目标/学习目标.md`

```md
# 学习目标

## 目标列表
- [ ] 目标A
- [ ] 目标B
- [ ] 目标C
```

- 这里只放 checklist
- 不写目标详情
- 目标顺序按当前优先级排列

## 单个目标详情页模板

文件路径：`项目名/学习目标/目标名/目标名.md`

```md
# {目标名}

## 目标描述
{这个目标具体要学什么}

## 创建日期
{YYYY-MM-DD}

## 状态
进行中

## 完成标准
{按 2-4 条写清楚，什么样算真正学会}

## 已有经验
{你现在会什么，不会什么}

## 必要性
{为什么必须补这个目标}

## 子项列表
- [ ] 子项1
- [ ] 子项2
```

- 重点写"能力成长"
- 不是纯技术实现说明
- `完成标准` 要可判断，不要空泛

## 内容风格规则

- 以第一人称成长视角为主
- 重点写：我为什么要学、我目前差在哪里、学到什么程度才算完成
- 不要写成官方文档口吻
- 不要堆太多术语
- 一页内容尽量控制在"短而清晰"

## 允许做的操作

### 创建新项目骨架
当项目不存在时，创建：项目目录、项目总览页、`学习目标` 目录、`学习目标.md`

### 为项目新增学习目标
创建 `学习目标/目标名/` 和目标详情页，并在 `学习目标.md` 中追加 checklist

### 更新已有页面
允许更新：项目状态、薄弱环节、学习目标列表、目标状态、子项列表、完成标准

### 追加，不重写
如果页面已存在，优先保留原内容，只补空缺字段或按用户要求追加/更新指定段落

## 不允许做的操作

- 不自动创建 README
- 不自动创建图片、附件、导图
- 不自动生成大量交叉链接
- 不把代码分析直接塞进总览页
- 不把项目页写成超长技术报告
- 不擅自改变目录结构
- 不新增本规范未定义的杂项文件

## 推荐默认目标数量

首次创建项目时，默认生成 3 个学习目标，优先从以下角度选：AI 辅助开发、代码质量/审查/调试、上线/运维/交付。如果用户明确给了目标，以用户目标为准。

## Skill 执行规则

1. 先读取本规范
2. 再判断项目是否已存在
3. 已存在则增量更新
4. 不得擅自改变目录结构
5. 不得新增本规范未定义的杂项文件
6. 所有新增页面都必须使用本规范模板
