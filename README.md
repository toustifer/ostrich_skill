# Oritch Skill Pack

This repository contains three exported skills:

- `brainstorming`
- `learning-project-summary`
- `learning-while-building`

## What This Repository Is

These files are skill definitions and companion assets for environments that support skill loading. They are not a standalone application package.

## Repository Layout

```text
skills/
  brainstorming/
  learning-project-summary/
  learning-while-building/
```

## Initialization

- If you want to use the note-oriented workflow, prepare a local markdown note application first. Obsidian is one option, but any tool that fits a local markdown workspace can work.
- Choose your own workspace root before first use.
- Keep note-storage paths unset in the public package. Replace placeholders such as `<your-workspace-root>` only in your private environment.
- If a skill needs an existing workspace spec, point it at your own local workspace root instead of editing the public package.

## Notes

- Personal paths and local-machine identifiers were removed from this export.
- Companion files such as `agents/`, `references/`, and `scripts/` are kept with the main skill when they are part of the workflow.
